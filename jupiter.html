SYSTEM ANALYSIS _

Jupiter est un plugin d'infrastructure pour Unreal Engine 5, conçu pour accélérer le développement de jeux de stratégie (RTS). L'architecture est entièrement modulaire, reposant sur une séparation stricte entre les données, la logique de contrôle et la représentation visuelle.

RTS CAMERA ARCHITECTURE

La classe APlayerCamera n'est pas un simple Pawn, mais un orchestrateur de systèmes. Plutôt que de tout gérer dans une seule classe géante ("God Class"), la caméra délègue ses responsabilités à des sous-systèmes dédiés (UCamera*System).

  • Movement System: Gère uniquement la translation, le zoom, et le "Edge Scrolling".
  • Selection System: Gère le Raycasting et la logique de sélection (Simple, Box, Multi).
  • Command System: Interprète les clics contextuels (Attaque, Move, Patrol) en fonction de l'entité sous la souris.

EDITOR TOOLS DETAIL

Le cœur de l'expérience développeur réside dans le UJupiterEditorPanel, qui héberge des outils spécialisés (Pages) pour la création de contenu in-game. Chaque page est une classe C++ distincte héritant de UJupiterPageBase.

TOOL A: UNIT SPAWNER (Page_UnitSpawn)

Outil principal pour le peuplement de niveaux. Il ne se contente pas de lister des acteurs, mais offre une gestion avancée de catalogue via UUnitsSelectionDataAsset.

  • Filtrage Dynamique: Système de recherche textuelle et filtrage par tags (via ApplyFilters()) pour naviguer rapidement dans de grandes bibliothèques d'unités.
  • Inspecteur Pliable: Un panneau latéral (InspectorContainer) permet de configurer les propriétés de spawn (Nombre d'unités, Formation initiale, Axe d'orientation) avant le placement.
  • Ghost Previz: Injection de dépendance vers UUnitSpawnComponent pour afficher un "fantôme" de l'unité sous la souris avant validation.

TOOL B: PATROL MANAGER (Page_Patrol)

Système de création de routes IA basé sur des Splines. Contrairement aux systèmes classiques points par points, celui-ci permet une édition visuelle fluide.

  • Spline Integration: Utilise UUnitPatrolComponent pour dessiner et modifier des courbes de Bézier directement dans le monde.
  • Route List & Detail: Interface maître-détail. La liste de gauche (EntryContainer) affiche toutes les patrouilles du niveau. Cliquer sur une entrée ouvre le PatrolDetailWidget pour ajuster les propriétés (Vitesse, Pause aux nœuds, Boucle).
  • Live Feedback: Le delegate HandleRoutesChanged assure que la liste UI reste toujours synchronisée avec les données du monde, même lors de suppressions/modifications externes.

AI & AUTONOMY _

RTS Controller Logic

La classe AAiControllerRts surcharge le contrôleur standard pour gérer les ordres vectoriels complexes. Elle intègre un système de patrouille hiérarchique avec gestion d'état (Attack, Patrol, Idle) et prise de décision locale (Engagement automatique si menace détectée).

Behavior Tree Integration

Le système expose des delegates Blueprint (FReachedDestination, FOnStartAttack) permettant aux Behavior Trees d'Unreal de réagir sans coupler le code C++ à la logique du BT.

MATH LIBRARY & FORMATIONS

Le positionnement des unités repose sur UFormationMathLibrary, une bibliothèque statique pure fournissant des algorithmes géométriques déterministes.

  • Geometric Patterns: Calculs vectoriels pour générer des offsets de formations standards (Ligne, Colonne, Carré, Triangle) et organiques (Blob).
  • Determinism: Pour la formation "Blob", un seed basé sur l'index de l'unité assure que la formation reste stable et reproductible (pas de jitter réseau).
  • Path Projection: Algorithme performant (CalculateClosestPointOnPath) pour projeter une position arbitraire sur le segment de patrouille le plus proche.
