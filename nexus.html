NEXUS FRAMEWORK CORE _

Nexus Framework est une couche d'abstraction structurée au-dessus de l'IOnlineSubsystem (Steam, EOS, Null). Il expose la logique réseau complexe via des nœuds asynchrones Blueprint-friendly.

ASYNC TASK SYSTEM

  • CreateSession: `UAsyncTask_CreateSession` gère automatiquement la destruction de l'ancienne session avant d'en créer une nouvelle, évitant les conflits.
  • FindSessions: Wrapper autour de `FindSessions`, exposant les résultats sous forme de liste structurée avec Ping et PlayerCount.
  • JoinSession: Gère la résolution d'IP et le ClientTravel.

ONLINE SUBSYSTEM WRAPPERS

  • Steam Integration: Support natif des lobbies Steam via `FOnlineSessionSettings`.
  • Event Driven: Le `AOnlineSessionManager` diffuse des événements globaux (`OnPlayerJoined`, `OnPlayerLeft`) pour l'UI, simplifiant la mise à jour des widgets comme le Lobby Screen.
  • Unique Net ID: Abstraction des PlatformIDs pour le banissement et le tracking cross-platform.

SESSION & HOST MIGRATION

La résilience des sessions est assurée par le UNexusMigrationSubsystem. En cas de déconnexion de l'hôte, le système élit automatiquement un nouveau leader sans interrompre la partie pour les clients restants.

  • Heir Election: Algorithme de sélection déterministe basé sur l'ancienneté (PlayerID) pour désigner le nouvel hôte.
  • State Recovery: Restauration automatique des GameStates et des PlayerStates après la migration.
  • Seamless Transition: Gestion transparente du Travel pour éviter les écrans de chargement inutiles.

ADVANCED MATCHMAKING

FILTERS & RULES ENGINE

Le système de matchmaking repose sur des `SessionFilterPresets` configurables en DataAssets. Il permet de définir des règles logiques complexes pour trouver la session parfaite.

  • Filter Rules: Prédicats logiques sérialisés (ex: `MapName == "Desert" && Ping < 50`).
  • Sorting Rules: Tri dynamique des résultats (ex: `Ascending(PlayerCount)`).
  • Flexible Query: Abstraction des requêtes spécifiques au sous-système (Steam Lobbies vs EOS vs LAN).

UTILITIES & HELPERS

SOCIAL INFRASTRUCTURE

MODULE: NEXUS CHAT

Un système de messagerie temps réel extensible, conçu pour les communautés persistantes. Il intègre des fonctionnalités d'accessibilité et de synchronisation avancées.

[ INSERT CHAT GIF ]
Typing Flow & History Sync
  • History Synchronization: Lors de la connexion (`OnRep`), les nouveaux joueurs reçoivent automatiquement les `N` derniers messages via `Client_ReceiveChatHistory`, assurant qu'ils ne manquent pas le contexte.
  • Text-to-Speech (TTS): Le `NexusVoiceComponent` peut lire vocalement les messages entrants (`Speak()`), améliorant l'accessibilité ou permettant des annonces système audibles.
  • Command Parsing: Support des commandes type IRC (/w user msg, /kick user, /reply).
  • UI Virtualization: Utilisation de UListView pour gérer des historiques de chat de milliers de lignes sans impact CPU.
  • Channel Routing: Système de dispatching dynamique pour isoler les flux de communication (Général vs Squad vs Party).
  • Blueprint Extensibility: Création de commandes custom (ex: `/emote`) via `RegisterBlueprintCommand()`, permettant aux designers d'étendre le chat sans C++.
  • Smart Hyperlinks: Le module `NexusLinkHelpers` parse automatiquement les URLs et permet d'insérer des liens interactifs riches (Items, Quêtes, GPS) cliquables dans le chat.
