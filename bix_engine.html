BIX ENGINE

C++ 20 FULL EDITOR SUITE CUSTOM REFLECTION

ENGINE ECOSYSTEM

Le Bix Engine est un environnement de développement complet. Au-delà d'un simple runtime, il intègre une suite d'outils d'édition natifs ("Page System"), un pipeline de build (BixAssetTool) et une architecture de données orientée performance.

SUITE OUTILS & EDITEURS

>>> EDITOR PAGE SYSTEM

Architecture UI fenêtrée basée sur le concept de Pages et Docking Spaces.

  • Fenêtres flottantes et layouts persistants (ImGui Docking).
  • Système de Focus et Raccourcis globaux.
  • Inspection en temps réel de n'importe quel UObject.

>>> VISUAL SPRITE ATLAS SLICER

Outil dédié au découpage de textures et à la génération d'animations.

  • Définition visuelle des régions de sprites.
  • Points de pivots et métadonnées d'animation.
  • Export optimisé en .atlas binaire.
[ INSERT ATLAS_TOOL.GIF ]

>>> ACTOR & PREFAB EDITOR

Éditeur visuel d'objets (Prefabs) pour la composition d'entités.

Components -> Visualization -> Serialization
  • Assemblage drag & drop de composants (Sprite, Collider, Script).
  • Modification hiérarchique : les changements sur le Prefab se propagent aux instances.
  • Sauvegarde en JSON lisible ou binaire optimisé.

>>> AUDIO CONTAINER SYSTEM

Système avancé de gestion des assets sonores.

  • Randomization : Variation automatique du Pitch et du Volume.
  • Virtualization : Gestion des voix via Miniaudio.
  • Looping & Streaming : Contrôle fin de la lecture mémoire.

GAMEPLAY FRAMEWORK

[ INSERT GAMEPLAY_DEMO.GIF ]

> INPUT SYSTEM (Action & Axis)

Le système d'input sépare la logique de jeu du matériel physique via une abstraction complète.

  • BindAction : Événements discrets (Saut, Tir). Gère les états Pressed, Released et Hold.
  • BindAxis : Valeurs continues (Mouvement). Normalise automatiquement les entrées Clavier/Gamepad entre [-1, 1].
  • Support du Hot-Plugging de manettes SDL3.

> COMPONENT DATABASE

[GFX] SPRITE ANIMATOR

Moteur d'animation 2D haute performance.

  • Lecture native des fichiers .atlas binaires.
  • Machine à états (Idle -> Run -> Jump).
  • Système d'événements (OnAnimationEnd, OnFrame).

[PHYS] BOX COLLIDER

Intégration profonde de Box2D.

  • Modes : Static, Kinematic, Dynamic.
  • Material : Friction, Restitution (Rebond), Densité.
  • Events : Callbacks C++ OnCollisionEnter / Exit.

[SFX] AUDIO SOURCE

Émetteur sonore 3D.

  • Spatialisation basée sur la distance (Min/Max).
  • Pooling automatique des voix pour éviter la surcharge CPU.
  • Support du Doppler Effect simulé.

[CORE] CAMERA

Système de vue flexible.

  • Gestion du Zoom et de l'Offset dynamique.
  • Helpers WorldToScreen et ScreenToWorld pour le UI.
  • Support de l'Aspect Ratio fixe ou adaptatif.

CORE & ARCHITECTURE

>>> UOBJECT DATAMODEL

La classe racine de tout le système de réflexion.

  • Apporte l'identité unique (UUID) à chaque entité.
  • Embarque le Transform (Position/Rotation/Scale).
  • Compatible avec la sérialisation automatique BPROPERTY.

>>> SCENE SYSTEM

Conteneur hermétique du gameplay.

  • Encapsule son propre monde physique Box2D.
  • Gère le cycle de vie des Acteurs (Spawn/Destroy).
  • Possède son contexte (Input, Render, Window) injecté.

>>> BINARY SERIALIZATION

Système de persistance haute performance.

  • Sauvegarde l'état complet du monde en binaire compact.
  • Utilise un système de Factory pour réinstancier les acteurs par leur nom de classe.
  • Supporte le versioning de données.

>>> CUSTOM MATH

Bibliothèque mathématique optimisée C++20.

  • Structures Vec3, Matrix4, Rotator (Euler) faites maison.
  • Système de Dirty Flags dans le Transform pour éviter les calculs de matrices inutiles.
